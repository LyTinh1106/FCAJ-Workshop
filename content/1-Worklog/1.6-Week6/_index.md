### Week 6 Objectives:

* Install AWS CLI and configure IAM account token and environment variables.
* Understand and enable AWS Security Hub to evaluate compliance and security posture.
* Implement structured tagging strategies on AWS resources via the Management Console and AWS CLI.

### Week 6 Achievements:

* **Configured Local Command-Line Environment**: Successfully installed the AWS Command Line Interface (CLI) to interface directly with AWS services.
* **Secured Local Authentication Context**: Configured IAM session tokens, access keys, and environment variables on the local system for secure API authentication.
* **Centralized Security Posture Management**: Enabled AWS Security Hub to continuously monitor and assess security posture across AWS accounts.
* **Evaluated Security Standards**: Reviewed overall security scores and findings against supported industry compliance standards (e.g., AWS Foundational Security Best Practices, CIS Benchmarks).
* **Implemented Standardized Tagging Strategy**: Enforced consistent metadata tagging on EC2 instances and other resources via both the AWS Management Console and AWS CLI.
* **Optimized Resource Discovery**: Configured resource filters based on tag keys and values to quickly locate specific workloads and environments.
* **Organized Resources with Groups**: Successfully created and configured AWS Resource Groups to simplify bulk operations, tracking, and organization of related resources.
