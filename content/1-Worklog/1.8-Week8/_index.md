### Week 8 Objectives:

* Understand the architecture and deployment process of Amazon Relational Database Service (Amazon RDS) for managed relational databases.
* Learn how to securely connect Amazon EC2 applications to Amazon RDS using VPC networking and Security Groups.
* Configure database backup, snapshot, and restoration mechanisms to improve data durability and disaster recovery readiness.
* Explore the core concepts of NoSQL databases through Amazon DynamoDB, including tables, items, attributes, and primary keys.
* Perform CRUD operations on Amazon DynamoDB using the AWS Management Console, AWS CloudShell, and AWS CLI.
* Integrate Python applications with Amazon DynamoDB using the AWS SDK (Boto3) to perform programmatic database operations.
* Compare relational and NoSQL database models, identifying suitable use cases for Amazon RDS and Amazon DynamoDB.
* Gain practical experience in deploying, managing, and integrating AWS-managed database services into cloud-native applications.

### Week 8 Achievements:

* **Deployed Secure Relational Database Infrastructure**: Built an Amazon RDS environment by configuring VPC networking, Security Groups, and DB Subnet Groups, ensuring secure communication between application and database layers.
* **Integrated Applications with Amazon RDS**: Successfully deployed a web application on Amazon EC2 and established reliable connectivity with an Amazon RDS database for data storage and retrieval.
* **Implemented Database Backup and Recovery**: Created database snapshots and validated restoration procedures, demonstrating the ability to protect and recover managed relational databases.
* **Developed Practical NoSQL Database Skills**: Created DynamoDB tables, managed items, and performed CRUD operations using the AWS Management Console, AWS CloudShell, and AWS CLI.
* **Built Programmatic Database Integration**: Configured AWS SDK (Boto3) and AWS CLI credentials to enable Python applications to interact with Amazon DynamoDB through API-based operations.
* **Compared Relational and NoSQL Database Models**: Gained practical understanding of the differences between Amazon RDS and Amazon DynamoDB, including their data models, scalability characteristics, and appropriate application scenarios.
* **Strengthened AWS Data Management Expertise**: Acquired hands-on experience in deploying, managing, securing, and integrating AWS-managed database services, providing a strong foundation for developing cloud-native data-driven applications.
